## Product Summary
Real-time offline social matching app, Korean market, Flutter.
Core loop: open app → match → venue suggestion → travel → BLE/UWB arrival confirmation → meet → rolling paper → figurine collectible.

## Critical User Flow Map
(See full output in agent response — stored here for agent memory continuity)

### Key Friction Points Identified
1. Permission wall (BLE + location + notification) at onboarding — HIGH severity drop-off
2. Venue travel commitment — user has to physically move — HIGH severity drop-off
3. Rolling paper feels awkward if meeting was mediocre — MED severity drop-off

### Trust Signals for Korean Market
- Real name / student email verification tier
- Match count + completion rate visible on profile
- Venue is a known branded place (not random)
- "이 매장 안전 파트너" badge on venue

### Big Five UX Pattern
- Frame as "취향 탐색" not "성격 검사"
- Max 2 questions per screen, swipe gesture
- Progress shown as figurine being "unlocked" not a bar
- Total: under 3 minutes target

### Retention Hook
- Rolling paper is collectible — recipient reads it later (delayed gratification)
- Figurine collection triggers completionist behavior
- Weekly "당신과 비슷한 사람이 근처에 있어요" push — scarcity framing
