# Waruru UX Flow — Research Output
_UX Researcher · 2026-03-26_

---

## 1. Core User Journey

**Persona:** 김지수, 29세, 강남 직장인, 퇴근 후 20:30

| # | Step | Action | System Response |
|---|------|---------|-----------------|
| 1 | Download & open | 카카오 소셜 로그인 | 온보딩 시작 |
| 2 | KYC | 셀피 + 신분증 앞면 촬영 | 실시간 OCR 인증 (목표 <90초) |
| 3 | Big Five test | 15문항 슬라이더 응답 | 성격 프로필 저장 |
| 4 | Interests + photos | 관심사 태그 선택 + 프로필 사진 2장 | 온보딩 완료 |
| 5 | Enter queue | "지금 만나기" 버튼 탭 → GPS 허용 | 실시간 매칭 큐 진입 |
| 6 | Match found | 상대 프로필 카드 노출 (이름·나이·성격 요약) | 양측 15초 수락 타이머 |
| 7 | Venue suggested | 중간지점 카페 1순위 + 대안 2곳 지도 표시 | OSRM 경로 + ETA |
| 8 | Navigate | 카카오맵/네이버맵으로 이동 | 앱 백그라운드 BLE 스캔 유지 |
| 9 | BLE arrival | 두 사람 모두 venue 반경 50m 진입 감지 | "둘 다 도착!" 알림 → 만남 시작 확정 |
| 10 | Rolling paper | 만남 종료 후 메시지 작성 | 상대에게 전송 + 디지털 피규어 수령 |

**Goal:** 앱 설치부터 첫 만남 성사까지 30분 이내
**Success Metric:** Step 1→9 완주율 35% (론치 4주차 기준)

## 2. Friction Points & Mitigation

| Step | Friction | Mitigation |
|------|----------|------------|
| KYC (Step 2) | "내 신분증 맡기기 싫다" 거부감 | 사용 목적 1줄 명시 + 즉시 삭제 보장 문구 |
| Big Five (Step 3) | 15문항도 길게 느낌 | 진행바 + "3분 소요" 명시, 건너뛰기 불가 (신뢰 핵심) |
| Queue wait (Step 5→6) | 매칭 안 되면 앱 종료 | 예상 대기시간 표시 + 백그라운드 푸시 알림 |
| 15초 수락 타이머 (Step 6) | 타이머 압박 → 충동 거절 | 상대 성격 요약 카드 즉시 노출, 타이머 색상 변화 |
| 이동 중 이탈 (Step 8) | "귀찮아서 안 감" | ETA 실시간 표시 + 상대방도 이동 중임을 암시 (정확한 위치 미노출) |
| BLE 도착 확인 (Step 9) | BLE 꺼짐 / 실내 약화 | 수동 "도착했어요" 버튼 폴백 항상 노출 |

## 3. 5 Critical Screens

| Screen | Purpose | Key Interactions |
|--------|---------|-----------------|
| **매칭 대기** | 큐 상태 + 기대감 유지 | 예상 대기 시간, 취소 버튼, 백그라운드 전환 허용 |
| **매치 카드** | 수락/거절 15초 결정 | 상대 Big Five 요약, 공통 관심사 태그, 타이머 |
| **장소 제안** | 의사결정 제거 | 지도 + 3곳 옵션 + 카카오/네이버 딥링크 |
| **이동 중 상태** | 이탈 방지 | 내 ETA + "상대방도 이동 중" 인디케이터 + BLE 상태 |
| **롤링페이퍼 작성** | 감성 루프 완성 | 텍스트 + 이모지, 전송 시 피규어 언박싱 애니메이션 |

## 4. Edge Cases

| Case | Handling |
|------|----------|
| 반경 내 venue 없음 | 반경 자동 확장 (500m → 1km) + "조금 더 이동" 안내 |
| 한쪽 취소 (이동 중) | 상대에게 즉시 알림 + 큐 재진입 원클릭 |
| BLE 비활성화 | OS 설정 딥링크 + "수동 도착 확인" 버튼 상시 노출 |
| 매칭 타임아웃 (15초 무응답) | 자동 거절 처리 + 큐 재진입 |
| 두 사람 ETA 차이 >15분 | venue 제안 전 경고 + 다른 상대 매칭 옵션 제공 |

---

## 5. Onboarding Drop-off Risks

| Risk | Drop-off Point | Fix |
|------|---------------|-----|
| KYC 거부 | Step 2 | 신뢰 지표 선노출 (인증 사용자 수, 보안 배지) |
| Big Five 피로 | Step 3 중반 | 문항 7번째에 "절반 왔어요" 마이크로카피 삽입 |
| 사진 없음 | Step 4 | 사진 없이 진행 불가 — 이유 명확히 ("상대방이 먼저 봅니다") |
| 위치 권한 거부 | Step 5 | 권한 요청 전 "왜 필요한지" 1화면 선설명 |
| 야간 큐 대기 무한 | Step 5→6 | 서울 5개 구 론치 집중으로 밀도 확보, 성수·강남 오후 7-10시 우선 |
