## 1. 디자인 시스템

### 1.1 브랜드 원칙

- **신뢰성 (Trust)**: 법령·조례를 다루는 플랫폼으로, 오류 없는 정보 전달이 최우선
- **명확성 (Clarity)**: 복잡한 절차와 법률 용어를 구조화된 레이아웃으로 단순화
- **전문성 (Authority)**: 공공기관·법조계 사용자를 위한 B2B SaaS 톤
- **효율성 (Efficiency)**: 전문가가 반복 사용하는 도구이므로, 최소 클릭으로 핵심 정보 도달

- 프라이머리 버튼 배경(`--color-primary-600`)에 흰색 텍스트: 대비비 7.2:1 (AAA 충족)

### 1.3 타이포그래피

#### 폰트 패밀리

```
--font-sans: 'Pretendard Variable', 'Pretendard', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Courier New', monospace;
```

- **Pretendard**: 법령 번호, 조항 텍스트, 한글 혼용 처리에 최적화된 가변 폰트
- **JetBrains Mono**: 법령 조문 번호, 코드형 식별자 (조합번호, 고유코드 등)

#### 타입 스케일

| 토큰명 | 크기 | 행간 | 자간 | 굵기 | 용도 |
|--------|------|------|------|------|------|
| `--text-display` | 32px | 40px | -0.5px | 700 | 페이지 메인 타이틀 |
| `--text-h1` | 24px | 32px | -0.3px | 700 | 섹션 제목 |
| `--text-h2` | 20px | 28px | -0.2px | 600 | 카드 제목, 모달 제목 |
| `--text-h3` | 16px | 24px | 0px | 600 | 서브 섹션, 테이블 헤더 |
| `--text-body-lg` | 16px | 26px | 0px | 400 | 주요 본문 |
| `--text-body` | 14px | 22px | 0px | 400 | 일반 본문, 설명 텍스트 |
| `--text-body-sm` | 13px | 20px | 0px | 400 | 보조 설명, 메타 정보 |
| `--text-caption` | 12px | 16px | 0.2px | 400 | 레이블, 배지, 타임스탬프 |
| `--text-code` | 13px | 20px | 0px | 400 | 법령 조문 번호 (mono) |

#### 법령 전용 타이포그래피 규칙

- 법조문 인용 시 `--font-mono` + `--color-neutral-800` + 좌측 보더(4px, `--color-primary-600`) + 배경 `--color-neutral-100`
- 법령명(예: 도시 및 주거환경정비법)은 항상 `font-weight: 600` + 언더라인 없이 컬러만으로 링크 구분
- 조항 번호(제X조, 제X항)는 `--text-code` 토큰 적용

```
│              │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 2.2 탑바 (Top Bar) 명세

- 높이: 64px, 배경: `--color-white`, 하단 border: 1px `--color-neutral-200`
- **좌측**: 정비나라 로고(24px) + 서비스명 텍스트 + 수직 구분선 + **조합 선택기(Dropdown)**
  - 조합 선택기: 현재 조합명 + 사업구분 뱃지(재개발/재건축 등) + 드롭다운 화살표
  - 드롭다운 열리면 조합 전체 목록 + 검색 입력 필드 표시 (사용자가 여러 조합 관리 시)
- **우측**: 알림 아이콘(뱃지 숫자) + 도움말 아이콘 + 사용자 아바타 + 이름 + 드롭다운

### 2.3 사이드바 메뉴 구조

배경색: `--color-primary-900`, 너비: 240px (접힌 상태: 64px)

```
──────────────────────────
  대시보드
──────────────────────────
  법령 관리
    ├ 법령 조회
    └ 법령 업데이트 알림
──────────────────────────
  절차 관리
    ├ 절차 플로우차트
    └ 단계별 진행 현황
──────────────────────────
  조례 비교
──────────────────────────
  일정 관리
    ├ 캘린더
    └ 마감기한 현황
──────────────────────────
  사례 검색
──────────────────────────
  설정
    ├ 조합 정보
    ├ 구성원 관리
    └ 알림 설정
──────────────────────────
```

#### 사이드바 항목 상태

- **기본**: 텍스트 `rgba(255,255,255,0.7)` + 아이콘 `rgba(255,255,255,0.6)`
- **호버**: 배경 `--color-primary-800` + 텍스트/아이콘 `rgba(255,255,255,0.95)`
- **활성(현재 페이지)**: 좌측 3px 보더 `--color-primary-500` + 배경 `--color-primary-800` + 텍스트 흰색 100%
- **하위메뉴 활성**: 들여쓰기 16px, 좌측 도트 `--color-primary-500`
- **접기 버튼**: 사이드바 하단 고정, 화살표 아이콘, transition 300ms ease

## 3. 화면별 와이어프레임 명세

### 3.1 대시보드 — 조합 현황 개요

#### 목적 및 사용자 목표
조합 담당자 또는 컨설턴트가 로그인 직후 현재 진행 중인 모든 중요 사항을 한 눈에 파악한다. 오늘 해야 할 일, 법령 변경 알림, 전체 진행률을 즉시 인지한다.

#### 페이지 헤더
- 제목: "대시보드"
- 부제: 조합명 + 사업 유형 뱃지(예: `재개발`) + 고시일(예: 정비구역 지정일)
- 우측: "절차 전체 보기" CTA 버튼 (Secondary)

#### 레이아웃 구조 (12컬럼 그리드)

**상단 KPI 카드 열 (4개, 각 3컬럼)**

```
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ 전체 진행률  │ │ D-day 마감   │ │ 법령 업데이트│ │ 미완료 서류  │
│   67%        │ │  D-12        │ │   3건 미확인 │ │  5건         │
│ 절차 18/27   │ │ 사업시행인가  │ │ 이번 달 개정 │ │ 이번 달 기한 │
└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘
```

각 카드 명세:
- 크기: 높이 120px, `--radius-lg`, `--shadow-sm`, 배경 `--color-white`
- 상단: 아이콘(20px) + 레이블 텍스트 (`--text-caption`, `--color-neutral-600`)
- 중단: 수치 (`--text-display`, `--color-neutral-800`)
- 하단: 부연 설명 (`--text-caption`, `--color-neutral-400`)
- "법령 업데이트" 카드는 미확인 건수 > 0 시 수치 색상 `--color-warning-600`으로 변경
- "D-day 마감" 카드는 D-7 이내 시 수치 색상 `--color-danger-600`

**중단 좌측 (8컬럼): 절차 진행 타임라인**

```
절차 진행 현황                             [전체 보기 →]
────────────────────────────────────────────────────────
  완료  ●─── 정비구역 지정      2024-03-15  완료
  완료  ●─── 추진위원회 구성    2024-06-20  완료
  완료  ●─── 조합설립인가       2024-11-10  완료
  진행  ◐─── 사업시행계획인가   2025-08-30  진행중 (67%)
  대기  ○─── 관리처분계획인가   2026-04-00  예정
  대기  ○─── 이주 및 철거       미정        대기
  대기  ○─── 착공              미정        대기
────────────────────────────────────────────────────────
```

- 각 행 높이: 48px
- 상태 아이콘: 완료 `circle-check` 초록, 진행중 `circle-dot` 파랑, 대기 `circle` 회색
- 진행중 행은 배경 `--color-primary-050`, 좌측 보더 3px `--color-primary-600`
- 현재 단계 행 클릭 시 → 해당 절차 상세 페이지로 이동

| 법령 업데이트 0건 | KPI 카드 수치 "최신" + 텍스트 `--color-success-600` |
| 절차 미시작 | 타임라인 첫 행만 표시, 나머지 희미하게 처리 |

### 3.2 법령 조회 화면

#### 목적 및 사용자 목표
재개발/재건축/소규모/모아주택 유형별로 관련 법령과 조례를 탐색하고, 특정 조항을 즉시 찾아 읽는다. 법령 개정 이력도 확인한다.

#### 레이아웃 구조

```
┌─────────────────────────────────────────────────────────┐
│ 법령 조회                                               │
│ [재개발] [재건축] [소규모정비] [모아주택] ← 탭 필터     │
├───────────────┬─────────────────────────────────────────┤
│               │  법령 상세 뷰어                         │
│  법령 목록    │                                         │
│  패널         │                                         │
│  (320px)      │                                         │
│               │                                         │
└───────────────┴─────────────────────────────────────────┘
```

#### 좌측 법령 목록 패널 (320px 고정)

- **검색 입력**: 상단 고정, 플레이스홀더 "법령명, 조항 번호 검색...", 아이콘 `search` 좌측 인라인
- **법령 카테고리 섹션**: 접기/펼치기 가능한 섹션 헤더

```
▾ 모법 (국가법령)
    도시 및 주거환경정비법              개정 ⚠
    도시재정비 촉진을 위한 특별법
    빈집 및 소규모주택 정비에 관한 특례법

    도시 및 주거환경정비법 시행령

▾ 지자체 조례 (서울특별시)
    서울특별시 도시 및 주거환경 정비 조례  개정 ⚠
    서울특별시 빈집정비 조례
```

- 각 법령 행 높이: 44px
- 활성 법령(현재 선택됨): 배경 `--color-primary-100`, 좌측 보더 3px `--color-primary-600`
- "개정 ⚠" 뱃지: 텍스트 `--color-warning-600`, 배경 `--color-warning-100`, `--radius-sm`
- 목록 행 호버: 배경 `--color-neutral-100`

#### 우측 법령 상세 뷰어

**상단 법령 헤더**

### 3.3 절차 플로우차트 화면

#### 목적 및 사용자 목표
재개발/재건축 등 사업 유형별 전체 절차를 시각적으로 파악하고, 현재 내 조합이 어느 단계에 있는지, 각 단계에서 필요한 서류와 관련 법령이 무엇인지 확인한다.

#### 레이아웃 구조

```
┌──────────────────────────────────────────────────────────┐
│ 절차 플로우차트                                          │
│ [재개발] [재건축] [소규모정비] [모아주택]  ← 사업유형 탭 │
│ 전체보기 / 주요단계만              ← 토글               │
├──────────────────────────────┬───────────────────────────┤
│                              │                           │
│   플로우차트 캔버스 영역      │  단계 상세 패널           │
│   (스크롤 가능, 세로 방향)    │  (360px 슬라이드인)       │
│                              │                           │
└──────────────────────────────┴───────────────────────────┘
```

#### 플로우차트 캔버스

- 스크롤: 세로 스크롤 (단계가 20단계 이상)
- 줌: 마우스 휠 + 핀치 줌 (75% ~ 150%)
- 미니맵: 우측 하단 고정 (120x200px), 현재 뷰포트 위치 표시

**단계 노드 디자인**

```
┌─────────────────────────────────────────────────────┐
│  ●  정비구역 지정                        완료        │
│     소관: 시·도지사                                  │
│     관련법: 도시정비법 제8조             2024-03-15  │
└─────────────────────────────────────────────────────┘
           │ (연결선 + 화살표)
           ▼
┌─────────────────────────────────────────────────────┐
│  ◐  사업시행계획인가                     진행중      │  ← 현재 단계 (파랑 테두리 2px)
│     소관: 시장·군수                                  │
│     관련법: 도시정비법 제50조            진행중      │
└─────────────────────────────────────────────────────┘
```

노드 상태별 스타일:
- **완료**: 배경 `--color-success-100`, 테두리 `--color-success-600`, 아이콘 `circle-check` 초록
- **진행중**: 배경 `--color-primary-050`, 테두리 2px `--color-primary-600`, 아이콘 `circle-dot` 파랑, 펄스 애니메이션 (그림자 크기 반복)
- **대기**: 배경 `--color-white`, 테두리 1px `--color-neutral-200`, 아이콘 `circle` 회색
- **불가(조건 미충족)**: 배경 `--color-neutral-100`, 테두리 1px 점선 `--color-neutral-400`, 아이콘 `lock` 회색

연결선:
- 완료 → 완료: `--color-success-600`, 2px 실선
- 완료 → 진행중: `--color-primary-600`, 2px 실선
- 진행중 → 대기: `--color-neutral-300`, 1px 점선

**분기 처리 (병렬 단계 표시)**
- 일부 단계는 병렬 진행 가능 (예: 이주 + 철거 동시 진행)
- 노드 2개를 같은 행에 배치 + 상단 "동시진행 가능" 레이블 + 점선 박스로 묶음

#### 우측 단계 상세 패널 (360px)

단계 노드 클릭 시 우측에서 슬라이드인 (300ms ease-out). 패널 내용:

```
[×] 닫기

사업시행계획인가
─────────────────────
상태: 진행중          예상 완료: 2025-08-30

관련 법령
  · 도시정비법 제50조  [바로가기 →]
  · 동법 시행령 제44조 [바로가기 →]

필요 서류 체크리스트
  ☑ 사업시행계획서
  ☑ 정관
```

#### 엣지 케이스

|------|-----------|

| 프린트/PDF 출력 | 단계별 체크리스트 포함 A4 레이아웃 별도 print CSS |

### 3.4 시도별 조례 비교 화면

#### 목적 및 사용자 목표
여러 시도의 도시정비 관련 조례 조항을 나란히 비교하여, 지역별 기준 차이(용적률, 인가 요건 등)를 파악한다.

#### 레이아웃 구조

```
┌──────────────────────────────────────────────────────────┐
│ 시도별 조례 비교                                         │
│                                                          │
│  비교 대상 선택:                                         │
│  [시/도 추가 + ]  [서울특별시 ×] [경기도 ×] [부산광역시 ×] │
│                                           (최대 5개)    │
│  비교 항목: [용적률] [건폐율] [조합설립 요건] [기부채납]  │
│             ← 항목 멀티셀렉트 드롭다운                   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  비교 테이블                                             │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

#### 비교 테이블 명세

헤더 행:
- 1열: "비교 항목" (고정, 200px)
- 2열~: 각 시/도 (나머지 너비 균등 분배)
- 1열은 수평 스크롤 시 좌측 고정(sticky)

```
┌──────────────────┬──────────────┬──────────────┬──────────────┐
│ 비교 항목        │ 서울특별시   │ 경기도       │ 부산광역시   │
├──────────────────┼──────────────┼──────────────┼──────────────┤
│ 재개발 용적률    │ 최대 300%    │ 최대 250%    │ 최대 280%  ▲ │  ← ▲ = 최고값
│ 재건축 용적률    │ 최대 270%    │ 최대 230%    │ 최대 260%    │
│ 조합설립 동의율  │ 토지 3/4     │ 토지 3/4     │ 토지 3/4     │
│ 분양신청 기간    │ 30~60일      │ 30~60일      │ 30일 이상  ▼ │  ← ▼ = 최저값
│ 기부채납 비율    │ 10% 이하     │ 8% 이하    ▼ │ 9% 이하      │
└──────────────────┴──────────────┴──────────────┴──────────────┘
```

셀 스타일:
- **최고값 셀**: 배경 `--color-success-100`, `▲` 아이콘 `--color-success-600` (유리한 방향 기준)
- **최저값 셀**: 배경 `--color-danger-100`, `▼` 아이콘 `--color-danger-600`
- **동일값 셀**: 기본 배경
- 셀 내 각주 아이콘(?) 클릭 시 해당 조례 원문 조항 툴팁 표시 (max-width 320px)

**조례 최신화 상태 행**
테이블 상단에 각 시/도별 조례 최종 업데이트 날짜 및 "최신" / "업데이트 필요" 뱃지 표시.

**내보내기 기능**
우측 상단: [Excel 다운로드] [PDF 출력] 버튼 (Secondary + 아이콘)

#### 엣지 케이스

| 상황 | 처리 방법 |
|------|-----------|
| 시/도 1개만 선택 | "비교를 위해 2개 이상의 시/도를 선택하세요." 안내 배너 |
| 조례 데이터 없는 지역 | 해당 셀 "준비 중" 텍스트 `--color-neutral-400` + 사선 배경 |
| 5개 초과 선택 시도 | 토스트: "최대 5개 시/도까지 비교 가능합니다." |
| 모바일 | 가로 스크롤 테이블 + 최초 항목 sticky 고정 |

### 3.5 일정 캘린더 화면

#### 목적 및 사용자 목표
조합 관련 모든 일정(인허가 마감, 총회, 서류 제출, 법령 시행일 등)을 캘린더에서 관리하고, 마감 임박 일정을 즉시 인지한다.

#### 레이아웃 구조

```
┌──────────────────────────────────────────────────────────┐
│ 일정 관리                   [월 보기 ∨]  [+ 일정 추가]  │
├─────────────────────────────────────────┬────────────────┤
│                                         │                │
│   캘린더 그리드 (월/주/일 보기)          │  일정 목록 패널 │
│                                         │  (280px)       │
│                                         │                │
└─────────────────────────────────────────┴────────────────┘
```

#### 캘린더 그리드 (월 보기 기본)

헤더: 이전 달 `<` + 연/월 제목 (`--text-h2`) + 다음 달 `>`

요일 헤더: 일 ~ 토, `--text-caption`, `--color-neutral-600`

날짜 셀:
- 크기: 가변 (그리드 1/7 × 행 자동)
- 최소 높이: 100px
- 오늘: 날짜 숫자 배경 원형 `--color-primary-600`, 텍스트 흰색
- 일정 표시: 셀 내 최대 3개 표시, 초과 시 "+N개 더" 링크

일정 칩(Chip) 디자인:
- 높이: 20px, `--radius-sm`, 텍스트 `--text-caption`
- **인허가/법정 기한**: 배경 `--color-danger-600`, 텍스트 흰색 (최우선 강조)
- **총회/회의**: 배경 `--color-primary-600`, 텍스트 흰색
- **서류 제출**: 배경 `--color-warning-600`, 텍스트 흰색
- **법령 시행일**: 배경 `--color-info-600`, 텍스트 흰색
- **기타**: 배경 `--color-neutral-200`, 텍스트 `--color-neutral-700`

D-day 배지:
- 마감 D-7 이내인 일정: 일정 칩 우측에 작은 `D-N` 텍스트 배지 (`--color-danger-600`)

#### 우측 일정 목록 패널 (280px)

날짜 셀 클릭 시 해당 날짜의 일정 목록 표시. 일정 없으면 "일정 없음 + 일정 추가 CTA".

## 6. 접근성 지침

### 6.1 색상 및 대비

- 모든 일반 텍스트: 배경 대비비 4.5:1 이상 (WCAG AA)
- 대형 텍스트(18px 이상 또는 14px bold): 3:1 이상
- UI 컴포넌트 및 그래픽: 3:1 이상
- 상태 정보는 색상 단독으로 전달하지 않음 → 항상 아이콘 또는 텍스트 레이블 병행

### 6.2 키보드 내비게이션

- 모든 인터랙티브 요소 Tab 키 포커스 지원
- 포커스 링: `outline: 2px solid --color-primary-600`, `outline-offset: 2px`
- 모달 열릴 시 포커스 트랩 (탭 순환)
- 모달 닫힐 시 트리거 요소로 포커스 복귀
- 플로우차트 캔버스: 방향키로 노드 간 이동, Enter로 상세 패널 열기
- 법령 조문 목록: 방향키로 항목 탐색

### 6.3 스크린 리더 지원

- 모든 아이콘 버튼: `aria-label` 필수
- 테이블: `<caption>`, `scope="col"` / `scope="row"` 적용
- 상태 변경 알림: `aria-live="polite"` (토스트, 로딩 완료)
- 플로우차트 노드: `role="button"` + 상태 텍스트 포함 `aria-label` (예: "사업시행계획인가, 진행중, 67% 완료")
- 비교 테이블 최고/최저 셀: aria-label에 최고/최저 여부 포함

### 6.4 인터랙션 피드백

- 모든 버튼: hover(150ms) + active(75ms) 전환
- 사이드바 접기/펼치기: 300ms ease transition
- 패널 슬라이드인: 300ms ease-out
- 모달 열림: 200ms scale(0.95→1) + opacity(0→1)
- 토스트 등장: 위에서 슬라이드다운 200ms, 자동 소멸 4초 후 fadeout

### 6.5 폼 접근성

- 모든 입력 필드: 명시적 `<label>` 연결 (placeholder는 레이블 대체 불가)
- 에러 메시지: `aria-describedby`로 필드와 연결
- 필수 입력: `required` 속성 + 시각적 `*` 표시 + 범례 안내
- 날짜 피커: 키보드 내비게이션 지원, 달력 팝업은 `role="dialog"`
