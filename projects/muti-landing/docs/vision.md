# Product Vision: muti

**One line:** 비개발자 창업자가 앱 아이디어를 자연어로 설명하면 AI 에이전트 팀이 자동으로 코드를 만들어주는 서비스의 랜딩 페이지

## The Real Problem

비개발자는 앱 아이디어가 있어도 실현할 방법이 없다.
- 외주 개발은 최소 500만~3000만원, 2~6개월 소요
- 노코드 도구(Bubble, Flutterflow)는 러닝커브가 높고 커스텀 한계가 있다
- AI 코딩 도구(Cursor, Claude Code)는 여전히 개발 경험이 필요하다

**핵심 인사이트:** "아이디어 → 동작하는 앱" 사이의 간극을 AI 에이전트 팀이 메워야 한다.

## Target User

**민수 (32세, 카페 창업자, 서울)**
- 카페 예약/포인트 앱을 만들고 싶지만 개발자를 구할 예산이 없음
- 노션에 기능 리스트를 정리해놨지만 실행 방법을 모름
- "내가 말만 하면 앱이 나오면 좋겠다"가 일상적인 생각
- 기술적 배경: 없음. 앱스토어에서 앱을 다운받아 쓸 줄만 앎

## Our Solution

**muti** — AI 에이전트 팀이 당신의 아이디어를 앱으로 만들어줍니다.

1. 사용자가 자연어로 앱 아이디어를 입력
2. AI CEO가 질문을 통해 요구사항을 구체화
3. AI CPO → Designer → Developer → Reviewer 에이전트 팀이 자동으로 기획, 디자인, 개발, 검수
4. 배포 가능한 코드가 결과물로 나옴

## MVP Features (3개)

1. **Hero + Value Proposition 섹션** — "아이디어를 말하면, AI가 만듭니다" 한 줄 메시지와 데모 영상/GIF로 서비스 핵심 가치를 3초 안에 전달
2. **인터랙티브 데모 / 아이디어 입력 CTA** — 랜딩 페이지에서 바로 아이디어를 입력해보는 체험형 인터페이스. 입력 → AI 응답 시뮬레이션으로 "이런 식으로 동작합니다"를 보여줌
3. **웨이트리스트 가입 폼** — 이메일 수집 + 얼리 액세스 대기열. 가입 시 순번 표시로 FOMO 유발

## North Star Metric

**웨이트리스트 가입 전환율 15%** (방문자 → 이메일 등록)
- 6개월 후 목표: 웨이트리스트 10,000명 확보

## Biggest Risk

**"AI가 정말 앱을 만들 수 있다"는 신뢰를 랜딩 페이지만으로 전달하기 어려움**
- 완성도 높은 데모 영상/시뮬레이션 없이는 "또 하나의 AI 마케팅"으로 치부될 위험
- 완화 전략: 실제 빌드 사례(Before/After), 라이브 데모 인터랙션, 소셜 프루프

## Platform

**Web** — Next.js 랜딩 페이지 (반응형, 모바일 퍼스트)

## Tech Stack (권장)

- Next.js 14 (App Router) + TypeScript
- Tailwind CSS + Framer Motion
- Supabase (웨이트리스트 이메일 저장)
- Vercel 배포
